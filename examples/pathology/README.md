# Pathology examples

The TAMIAS repository deliberately does not ship full whole-slide images
in `examples/`. Production WSIs are large (often hundreds of MB at
lo-res), and fetching them at install time would slow every developer
checkout.

Instead this folder ships:

- A small **synthetic H&E patch** (`synthetic_he_512.png`, 512 × 512,
  CC0). Procedurally generated by `scripts/build-pathology-sample.mjs` —
  no real patient data, no third-party copyright. The pink/purple
  staining and circular nuclei give the Pathology mode something
  realistic to render so users can verify the viewer + brush + tile
  inference pipeline end-to-end without first downloading a real WSI.
- A reference **pathology manifest** (`tissue_mask.json`) exercising
  every schema field validated by the parser.
- Pointers to public-domain WSIs you can download and try locally.

## Trying the bundled sample

1. Open the app and switch to **Pathology** mode (Microscope toggle in
   the header).
2. Drop or pick `synthetic_he_512.png` into the Slide picker. The
   distance ruler will read in pixels (this PNG carries no MPP metadata
   — the sidecar `synthetic_he_512.json` declares 0.5 µm/px so a
   downstream analysis tool that reads it can render in microns).
3. Optionally pick the `tissue_mask.json` manifest plus your own ONNX
   to exercise the inference path end-to-end.

## Converting SVS / NDPI to OME-TIFF

`v0.6.0` does not yet ship OpenSlide-WASM, so vendor-format slides
(Aperio SVS, Hamamatsu NDPI) need an offline conversion. Two options:

**bioformats2raw + raw2ometiff** (recommended for whole TCGA cohorts):

```sh
bioformats2raw input.svs out.zarr
raw2ometiff out.zarr input.ome.tif --compression=jpeg
```

**QuPath** (GUI, nice for one-off slides):
File → Export → OME-TIFF.

## Running the bundled tissue-mask manifest

`tissue_mask.json` is a reference manifest — it does **not** ship with
its own ONNX file because there is no public-domain pathology model that
we'd reuse without verification. To exercise the pipeline end-to-end:

1. Train or export any 3-channel input → 1-channel output ONNX patch
   model (HoVer-Net, CLAM, simple U-Net, etc.).
2. Adjust the `name`, `mpp`, `patch`, `stride`, and `output` fields in
   `tissue_mask.json` to match your model.
3. Load the slide + ONNX + manifest in the Pathology mode and click
   "Run on full slide".

The `inference` panel in the app shows progress patch-by-patch and the
result is overlaid on the slide using the colours declared in the
manifest's `output.colors` (or auto-assigned from the default palette).
